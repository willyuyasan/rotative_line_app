#DB changes
all the DB changes has to be made in .env file
some especial postgres db from NEON or vercel, requires that you feed with DB_URL = DATABASE_URL_UNPOOLED instead of credentials
use:
php artisan migrate:fresh
php artisan db:seed
or:
php artisan migrate:fresh --seed



#follow the get started info of filament
#create the admin user
php artisan make:filament-user

#add the spatie component to manage roles and permissions (follow the spatie doc)
https://spatie.be/docs/laravel-permission/v6/installation-laravel
composer require spatie/laravel-permission --ignore-platform-reqs


#Refresh de seeder to add role to users
#midify the DatabaseSeeder as here is showed
php artisan migrate:fresh --seed


#Create the resources for managing users
php artisan make:filament-resource User --view

php artisan make:model Role -m
php artisan make:model Permission -m
php artisan make:filament-resource Role --view --generate
php artisan make:filament-resource Permission --view --generate


#Creating models policies to control role access to the views
php artisan make:policy UserPolicy --model=User
php artisan make:policy RolePolicy --model=Role
php artisan make:policy PermissionPolicy --model=Permission

and modify the permisions over the actions in the policy folder

php artisan cache:forget spatie.permission.cache 
php artisan permission:cache-reset
php artisan cache:clear

finally configure a gate to allow a super admin
go to: 
Providers/AppServiceProvider.php
